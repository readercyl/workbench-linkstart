# 对话存档·回流模板

> 来源：日报 SKILL.md v2.0 · 第二步「整理昨日存档」中回流声明的格式。AI 执行回流时按需读取。

## 默认格式

大多数天，无新认知时使用：

```markdown
<!-- 回流：已检查，0 条新认知 | 原因：[本轮全部为系统运维/纯执行确认/闲聊等，不产生独立知识] | 0 处个人认知更新 -->
```

## 扩展格式（回流清单）

有新认知但未立即编译时使用。回流清单表只在有明确可命名的认知时填写——不是每个对话存档都需要。

```markdown
<!-- 回流清单
| 来源主题 | 可提炼点 | 建议卡片名 | 状态 |
|---------|---------|-----------|------|
| 主题5 | 249:1输入输出比下缓存价决定一切 | 缓存命中率是模型性价比的唯一变量 | ⬜ 待编译 |
| 主题8 | 治理的本质是砍机制不是加机制 | 待提炼 | ⬜ 待编译 |
-->
<!-- 回流：已检查，2 条可提炼认知待编译 | 0 处个人认知更新 -->
```

## 💎 内联标记约定

在主题正文写作时，如果某段包含可提炼为知识卡片的认知，AI 可在段落末尾加 `💎` 标记。不影响可读性，让后续知识编译扫描存档时能用 `grep '💎'` 快速定位候选段落。每 2-3 个主题约出现一次，不需要每段都标。
