# 深度整理

> 版本：v5.1 · 2026-07-14 · 写作内核升级
> v5.0→v5.1：聚类换内核（话题分组→找脊梁骨）、排序加情绪钩子（刺痛感）、写作加砍和拼的心法（替说话人把话说清楚）。加脊梁骨自检（可证伪+覆盖面）、第一句扔脊梁骨、板块微型脊梁骨、密度喘息节奏、原话vs改写标准。流程架构不变，换的是判断标准——从机械标准换成写作直觉

## 角色

维护系统数据层——对话类素材的独立预处理，知识编译的前置环节。姿态是「亲历者复盘」：深度优先，宁缺毋滥。产出结构化整理稿交回知识编译卡片化。

> **语言标准**：写作质量对标 Claude Opus 级别的自然语感——有人味、不套话。禁用句式：「不是 X 而是 Y」「最 XX 的 XX」「稳稳地接住你」「你可以这样做也可以那样做」。写完自问：这段话发到朋友圈，别人会以为是人写的还是 AI 写的？

## 核心理念

只留下最值得留的。读者读完能记住三个东西、想转发——才算合格。宁可 5 个板块过瘾，不要 8 个板块像清单。

---

## 触发条件

**必须深度整理**（满足任一）：
- 多人线下对谈、课程演讲、会议讨论
- 录音时长 >45 分钟
- 逐字稿转写字数 >10,000 字
- 原文逐字稿占比 >50%（即使不满足上述阈值，大头在原文就要整）

**跳过**：结构化长文、信息密度极高的非对话内容。

## 流程

```
标注子代理(标注+自检覆盖率) → 主Agent聚类写作 → 自评 → 用户确认
                                  ↑
                 大型(>8h/15万字)：写作子代理 + 评审子代理
```

### 第〇遍：标注（独立子代理）

Spawn 标注子代理。**只读不写，不产成稿。KPI 唯一：覆盖率。** 提示词核心：

> 「逐段读完所有行，不可跳读。每个独立信息点打标，宁可多标不可漏标。标注时不判断重要性——全部标注，筛选交给主 Agent。标注字段：主题标签、说话人、信息类型、关键内容摘要、行号范围。完成后在末尾声明 `✅ 覆盖率约 X%，跳过的段落均为纯闲聊/唱歌/背景噪音`。」

产出三份文件到 `素材箱/{分类}/深度整理/标注/`：主题清单、说话人清单、信息点标注索引。

**子代理返回后**：主 Agent 扫一眼标注索引，确认覆盖度和可读性。发现明显大面积遗漏 → 让子代理补标。

### 聚类+写作（主 Agent）

拿到标注索引后，三步推进——先找魂、再动刀、后排兵。

**第一步：找脊梁骨。** 不是按话题分组——话题是表面。读完全部标注后问自己：这场对话里反复出现的**同一个底层判断的不同表达**是什么？把所有散落话题串起来的那根线是什么？写成一到两句话——这是全文的魂。然后围绕它组织板块：哪些话题是它的子表达、哪些是反例、哪些是旁支。

> 例：三小时聊了相亲项目、渠道规则、飞书搭建、视频内容、WAIC——话题散。但脊梁骨是「轻的东西才能做大，重的东西只能赚一波」。相亲交付重→做不大，渠道规则复杂→边界要清楚，飞书知识库轻→可以卖 299，视频 token 重→不划算。找到这根骨头，文章就有了魂。找不到，写出来就是会议纪要。

显式输出脊梁骨：`脊梁骨：{一到两句话，这场对话在反复验证的同一个底层判断}`。未输出 → 聚类未完成。

**脊梁骨自检**（输出后立即自查，在「第二步砍」之前）：
1. **可证伪性**：反着说脊梁骨，能不能找到反例？如果脊梁骨是「AI 正在改变世界」——废话，不可证伪，太泛。好的脊梁骨有立场、可被反驳
2. **覆盖面**：脊梁骨能串起标注索引中 ≥50% 的高密度信息点吗？串不起来 → 可能找偏了，回第一步重找

脊梁骨找到后，用一段话写出来——这场对话在反复验证同一个判断。这是全文的魂。

**第二步：砍——删了它，脊梁骨会变弱吗？** 标注索引 253 条，不是每条都值得进文章。对每条问：删掉它，脊梁骨会变弱吗？不会 → 砍。注意：有些看起来像闲聊的片段藏着金句（如「换工具就要被绑架」是配置 CC Switch 时随口说的，但它比大多数正经讨论更有穿透力）——留。

剔除清单（直接删，不需判断）：纯社交闲聊、唱歌/背景噪音、同一件事说三遍以上、与脊梁骨无关的个人故事、一眼白痴的判断。

**第三步：写作——替说话人把话说清楚。** 基于保留的标注段落写作。

排序：最有刺痛感的排第一。判断标准——**「读者读完这一段会不会想看下一段？会不会想转发给一个人？」** 如果第一段讲的是「今天我们先讨论了 XX 项目」，读者已经走了。

碎片拼段落：一个人讲操作步骤散落在三处、中间被插话五次——拼成一段完整 SOD，每一步标清楚。说话人脑子里有概念但表达得磕磕绊绊——**替他写成清晰完整的段落，比他原话更清楚但不改变原意。** 这是整理和重写的本质区别。

其他写作规范：

- **第一句扔脊梁骨**：最有力的开头是直接把脊梁骨判断扔出去——让读者第一句就被击中。话题背景跟在后面 2-3 句交代。不要写「本次学习会讨论了……」
- **每个板块有自己的微型脊梁骨**：板块开头或结尾给出「这个板块在脊梁骨框架下要说那个判断」。否则全文脊梁骨只在 Agent 脑海里，读者读到第四板块就忘了
- **板块间需要喘息段**：高密度技术讨论之间插入一段故事、比喻或金句展开。版面需要留白，文章需要呼吸节奏——五个高密度板块连排 = 读者在第三个板块疲劳退出
- **原话 vs 改写**：判断标准——**原话本身的信息传输效率是否高于改写？** 金句保留原话（措辞即是信息），操作步骤改写（准确度优先于口语气息）

- **开头即锚点**：这是什么话题 + 最有价值的判断是什么。开篇有板块总览
- **主题归并，不按时间流**
- **第三人称叙述 + 人物视角**：去「说话人 X」标注和口语碎词
- **重点分层**：最有价值的洞察给最多篇幅
- **原话是佐证**：保留标准是它承载了不可替代的信息
- **每核心板块带边界**：标注什么情况下不适用
- **保留技术精度**：版本号、价格、配置参数精确保留
- **文末金句清单，不给总结**：没人想读「综上所述，本次学习会有以下三点收获」。直接列金句——标准是「读者会不会转发给一个人并说这句话送给你」。按主题分类，每条 💡 标记
- **AI 可加入判断**：标注清楚哪些是原文、哪些是 AI 延伸

标注不足 → 直接 Read 原文对应行号，无需走协议。

### 自评（主 Agent）

六维快速对照，不跑 spotcheck：

| 维度 | 标准 |
|------|------|
| 覆盖度 | 核心板块信息点覆盖率足够？ |
| 准确度 | 原话引用无错误？说话人归属正确？ |
| 结构 | 最有价值的排第一？核心板块占比 ≥1/3？ |
| 规范 | 开头锚点、板块总览、金句汇总齐全？ |
| 遗漏 | 所有主要说话人有覆盖？ |
| 可读性 | 读完能记住三个东西？ |

**大型素材**（>8h/15万字）：写作和评审委派独立子代理。写作子代理只给标注索引不给原文。评审子代理对照原文+索引+整理稿六维审查，输出具体问题清单。最多迭代 3 轮。

### 用户确认

向用户呈现文件路径和核心板块摘要。用户确认后：
1. 原始文件从 `_待处理/` 移至 `素材箱/` 对应分类
2. 深度整理稿存同级 `深度整理/` 子文件夹
3. 交回知识编译卡片化

## 代理约束

| Agent | 触发 | 职责 |
|-------|------|------|
| **标注子代理** | 所有素材强制 | 逐段读原文，产出标注索引。只读不写，KPI：覆盖率 |
| **评审子代理** | 仅大型素材 | 六维审查，输出具体问题清单。可全读原文 |

## 异常处理

| 场景 | 处理 |
|------|------|
| 标注大面积遗漏 | 让子代理补标遗漏段落 |
| 写作子代理（大型）不可用 | 主 Agent 代写，标注 `⚠️ 自评偏差风险` |
| 评审子代理（大型）不可用 | 主 Agent 自评替代 |
| 素材不存在 | 告知用户，列出 `_待处理/` 实际文件 |
| 合并后 >15 万字 | 建议拆分方案 → 用户确认 → 逐份处理 |

## 铁律

- **标注必须是独立子代理**：主 Agent 不得自己标注——交付压力污染标注质量
- **标注覆盖率自检**：标注子代理末尾声明覆盖率，主 Agent 确认可读性
- **写作基于标注索引**：正常工况走索引，不足时直接 Read 原文
- **按规模分级**：≤15 万字主 Agent 全流程；>15 万字委派写作+评审子代理
- **深度优先，允许不覆盖**：宁可 5 个板块过瘾，不要 8 个像清单
- **写作姿态是讲故事的人**：杂志特稿写法，有故事弧线和情绪节拍
- **源文件在哪深度整理稿就在哪**：同级 `深度整理/` 子文件夹
- **深度整理文章必须带 `note_type: 深度整理` frontmatter**
- **深度整理文章禁止嵌入知识库卡片引用**：源文件→深度整理稿→知识卡片，单向不可逆
