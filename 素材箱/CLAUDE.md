## 铁律

- 原始文件内容**绝对不能修改**
- 可改文件名以更好反映内容
- `_待处理/` 保持空——进来就编译移走

## getnote 视频日记导出格式

**文件命名**：`YYYY-MM-DD-第N天-主题关键词.md`，与目录内已有文件保持一致。
- 日期：日记的录音日期（从 getnote 标题中的「YYYY年M月D日」提取，统一为 `YYYY-MM-DD`）
- 第N天：顺延上一个已有文件的天数（如已有第127天 → 新文件为第128天）
- 主题关键词：从 getnote 标题中「日记：」后的部分提取，保持简洁

**内容格式**：从 getnote API 导出时**不加 frontmatter、不写 API 元数据**（note_id、note_type、tags 等）。保留以下全部内容节：

```markdown
# {标题}
> 来源：getnote「我的视频日记」知识库
### 📑 智能总结
{getnote content，含录音信息和录音总结，原样保留}
### 📅 章节概要
{原样保留}
### ✨ 金句精选
{原样保留}
### 📋 待办事项
{原样保留}
## 原语转写
{audio.original 全文，原样保留}
```

此格式 2026-06-06 与用户确认锁定。

**硬门控**：每次导出后必须立即运行保护脚本，不通过 = 阻断后续流程：

```bash
bash AI协作工具/skills/数据层/知识编译/视频日记导出保护.sh <导出文件路径>
```

脚本检查：原语转写节存在、有实际内容、含说话人标记。任何一项不通过 → 重新导出，不得进入编译。
此规则 2026-07-04 确立，根因：AI 多次在 getnote 导出时漏掉 `audio.original` 原语转写。
