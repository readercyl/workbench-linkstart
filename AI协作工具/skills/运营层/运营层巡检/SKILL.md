# 运营层巡检

> 版本：v2.0 · 2026-07-14 · 维护系统重构
> v1.7→v2.0：**全量重构**。① 新增角色自述；② 三模式压缩（日常轻扫表格保留、巡视/任务流程精简）；③ 旧引用全量修正（知识库体检→数据层体检、运转审计→框架层审核、守序→运营层巡检）；④ 核心原则+内环六步去重合并；⑤ 外环自我进化压缩；⑥ 新增「审完怎么修」路由表

## 角色

维护系统三层之一，管每天的运营健康。5 项纯机械检查 <10 秒，不动 AI 判断。发现问题自己修路径/格式/计数类确定性 bug，修不了或需语义判断的上报。上下游：发现知识管线异常→通知「数据层体检」；发现 Skill/配置问题→委托「框架层审核」。

## 三模式

| 模式 | 触发 | 做什么 |
|------|------|--------|
| **日常轻扫** | Cron 每天 18:03 · 用户说「轻扫」 | 5 项机械检查。全通过静默，异常当场修 |
| **巡视** | 用户说「巡检」· 编译收口后 · 框架层审核后 | 扫描信号池→诊断→处置 1 项→验证→记录 |
| **任务** | 用户给定停止条件 | 循环修复直到条件达成，独立子代理验证 |

> 日常轻扫同一天已执行且全部通过 → 跳过不重复。巡视/任务不静默，必定给用户摘要。

### 日常轻扫

| # | 检查项 | 命令 | 异常证据 |
|---|--------|------|---------|
| 1 | 编译卡片未收口 | `grep -rl "moc_added: false" 02-知识卡片/知识卡片/` | 列出卡片名 |
| 2 | 脚本语法 | `bash -n` 扫描 `数据层/知识编译/` 下所有 .sh | 损坏文件路径+行号 |
| 3 | 软链接断链 | `find -L .claude/skills -type l ! -exec test -e {} \; -print` | 断链完整路径 |
| 4 | Git 未提交 | `git status --porcelain \| wc -l` > 20？ | 文件数，>50 列文件名 |
| 5 | 新卡未入 MOC | 近 3 天新卡是否在 MOC 中有引用 | 未归入的卡片名 |

> 只修确定性 bug（格式/路径/计数/symlink）。修不了就记 `.sh-state.md` 等下次巡视。

### 巡视

扫描信号池 → 诊断排优先级 → 每轮处置 1 项 → 独立子代理对抗验证 → 写回 `.sh-state.md`。

| 信号 | 阈值 | 优先级 |
|------|------|:--:|
| 编译卡片未收口 | 非空 | 🔴 |
| 脚本冒烟测试失败 | bash -n 报错 | 🔴 |
| 软链接断链 | 存在 | 🔴 |
| `.network_index.json` 过期 | >24h | 🟠 |
| 调用记录 count-状态不匹配 | 需对比 | 🟡 |
| S1 最近扫描有 🔴 | 上次 exit 1 | 🟠 |
| 周复盘有 pending 行动项 | 存在 | 🔵 |

> 同信号出现 ≥2 次才升级（单次可能是瞬态）。🔴 首次即处置。

每轮取最高优先级 1 项。修复：简单（Edit/Bash）→自己修；复杂（多文件/需语义）→独立子代理；不确定→写 `.fix-proposals/` 等用户。

### 任务

循环修复→验证，直到停止条件达成。最多 2 轮——超过则升级用户裁决。

## 审完怎么修

| 路由目标 | 触发条件 |
|---------|---------|
| **自己修** | 路径、格式、计数、symlink 类确定性修复 |
| 「知识编译」→ 编译收口.sh | 卡片未收口 |
| 「数据层体检」 | S1 有致命项、网络索引过期、调用记录异常 |
| 「工具自修正」 | 脚本语法错误、Skill 逻辑缺陷 |
| 「框架层审核」 | 流程设计缺陷、Skill 累积 ≥3 条纠正 |
| 「日报」 | 周复盘有 pending 行动项 |
| 标 ℹ️ 等用户 | 不确定的修复、Skill 3 天内同一问题再犯 |

## 自我进化

- **自主发现**：同一问题 `.sh-state.md` 处置 ≥3 次 → 根因分析 → 向用户提议修正 SKILL.md
- **用户纠正**：单次→工具自修正；累计 ≥3 条→委托框架层审核；3 天内再犯→标记自修正无效

## 铁律

- **制造者 ≠ 检查者**：修复和验证永远不是同一个模型
- **状态落盘**：每轮写 `.sh-state.md`，下一轮从此继续
- **不越界**：只修确定性 bug。语义判断和逻辑变更只提议
- **不抢 token**：每轮最多处置 1 项
- **全静默或带证据**：全通过→用户无感知。有异常→附文件名+路径+行号
