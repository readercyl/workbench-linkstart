# 日报

> 版本：v2.2 · 2026-07-14 · 回流收口
> v2.1→v2.2：第二步末尾加「回流收口」——整理存档后检查回流声明中的待编译条目，有则自动调知识编译，补上回流→编译的断裂环节

## 角色

维护系统运营层——每日跨日快照。先整理昨日存档再生成日报。Hook 触发→幂等自检→存在则跳过。

## 流程

### 第一步：幂等自检

`00-运转日志/日报/${TODAY}*.md` 存在 → 静默跳过。不存在 → 继续。

### 第二步：整理昨日存档

**这是日报的前置步骤——整理后的存档是日报的数据源。**

读昨日对话存档文件（`00-运转日志/对话记录/${YESTERDAY}-*.md`），执行整理：

1. **合并碎片**：同一主题的多次追加合并为一个主题模块
2. **提炼主线**：写 `一句话总览`——昨天所有主题在表达的「同一件事」
3. **写 AI 摘要**：主题、用户状态、AI 洞察、关键决策、产出
4. **写 frontmatter**：`topics`、`decisions`、`cards_produced`、`tags`
5. **回流声明**：新认知→知识库、新理解→人物档案、`💎` 标记段落→编译候选
6. **打标记**：`consolidated: true` + `consolidated_at: YYYY-MM-DD`
7. **跑调用记录**：
   ```bash
   bash AI协作工具/skills/运营层/日报/更新调用记录.sh <存档文件路径>
   ```
8. **质量评估**：委派独立子代理评估整理质量——内容是否完整覆盖了存档维度（决策/认知/AI洞察/反思）、信息是否有价值、有没有遗漏关键思考。子代理只评不改，结果返回后主 Agent 对照调整。评估要点：不求格式完美，求信息不丢。
9. **回流收口**：检查回流声明中是否有 ⬜ 待编译条目，以及存档正文中是否有 💎 标记段落——有则调用「知识编译」Skill 编译该存档文件。回流声明写了就必须消费，不留隔夜债。无待编译条目 → 跳过。

> 整理模板见 `references/整理模板.md`，回流模板见 `references/回流模板.md`。

### 第三步：收集昨日信息

**先跑数据脚本**：

```bash
bash AI协作工具/skills/运营层/日报/日报数据.sh
```

脚本输出 JSON，包含：卡片数、个人说明书数、存档/日报数、待处理数、昨日 log 编译记录、昨日 git commits。

**AI 补充**：从昨日整理后的存档提取关键信息，读取本周已有日报续写演化轨迹。

### 第四步：撰写报告

```markdown
---
type: 日报
date_covered: YYYY-MM-DD
summary: [1-2 句，不超过 200 字，概括昨日核心变化。]
---

# 日报 · YYYY年MM月DD日

> 昨日（MM月DD日）一句话定调：[昨天相比前天发生了什么变化。]

## 一、知识库状态

| 指标 | 数值 | 较前日 |
|------|------|--------|
| 知识卡片总数（02） | N | +/- |
| 个人说明书文件（01） | N | +/- |
| 对话存档 | N 篇 | +/- |
| 日报 | N 篇 | +/- |
| 待处理 | N | +/- |

**昨日状态**：[从整理后的存档 AI 摘要提取]

## 二、本周演化轨迹

| 日期 | 类型 | 主题 | 强度 |
|------|------|------|------|
| MM/DD | ... | 极低/低/中/高/极高 |

强度：极高(>5决策/>20增量) / 高(3-5/10-20) / 中(1-2/1-10) / 低(0决策有小改) / 极低(静默)
类型标签：🔧系统 / 📚知识 / 💼业务 / 📖学习 / 🤝社交

## 三、跨日提示

> 命中触发清单→写1-3条洞察；未命中→整节跳过。

触发清单：
- 演化轨迹连续 ≥3 天「极高」强度
- 卡片数较前日负增长
- 用户状态连续「高强度」但当日静默
- 前日关键决策 >5 且当日=0
- 本周无「💼业务」类型天

```

### 第五步：写入

文件名：`YYYY-MM-DD-日报-主题关键词.md`，写入 `00-运转日志/日报/`。

### 第六步：错字扫描

```bash
bash AI协作工具/skills/运营层/_工具/错字扫描.sh <日报文件路径>
```

exit 1 → 逐处修正 → 重跑直到通过。

## 铁律

- **先整理再出报**：昨日存档不整理 = 日报没有数据源，必须先做第二步
- **hook 触发即做**：收到 hook 注入的指令就直接执行，不问用户
- **唯一幂等依据是文件存在**：`ls ${TODAY}*.md` 匹配到 → 跳过
- **不重复对话存档**：日报只写跨日数据。关键决策/认知归存档
- **数据收集走脚本**：机械数据跑 `日报数据.sh`，AI 不手动数
- **回流必收口**：整理完存档必须检查回流声明，有 ⬜ 待编译条目或 💎 标记 → 调知识编译。回流链条不能断在「写了但没人执行」
