
### 标题命名规范

标题是 AI 语义检索和编译时卡片关系判断的**唯一语义索引**——读完标题就能判断这张卡是否跟某个问题相关。

**命名原则**：
- **标题即核心论点**：不用「XX方法」「XX模型」「XX原则」模板化命名，用一句话核心观点
- **双段式优先**（`A—B`）：前段抛出具体观点，后段解释或举例。如「先执行再优化-完成比完美重要但不是盲动」
- **具体 > 抽象**：「单点切入-一个渠道500元比全渠道打包更容易成交」而非「渠道策略」
- **一个标题一个点**：不用「与」「和」「及」并列式，不用问句

**约束**：
- 8-25 个字符，信息密度够但不过长
- 不用「方法」「模型」「原则」「框架」等模板化后缀——如果非用不可，前面必须有具体观点

**反例→正例**：
- ❌ `用户分层五层模型` → ✅ `用户分层不是分人群-五层按付费意愿分`
- ❌ `48小时启动原则` → ✅ `48小时启动原则-超48小时立项永远动不了`
- ❌ `选择即放弃` → ✅ `选择即放弃-不选同规模选项是内耗根源`

**旧标题渐进更新**：导入自己的旧卡后，不做低回报的全量重命名。检索命中某张旧卡时，再按新规范修正标题并同步引用。

**Frontmatter**：`source_type` 八种取值如上行管道列表。`status` 四阶段——`seed`（默认）→ `growing`（调用≥3或入度≥2或多来源验证）→ `evergreen`（调用≥10或入度≥5）→ `dormant`（调用=0且入度=0，体检标记）。升级由体检 Skill 自动执行。

**正文**：一个点一张卡，`[[链接]]` 自然嵌入句子中。链接追求语义质量而非数量——宁缺勿凑，不因「必须链一张」而挂通用方法论万金油。中文写作中文标点。

**相关链接**：列出正文引用的所有 `[[链接]]`，每卡独占一行。只用裸 `[[卡片名]]`，不使用类型前缀。卡片之间只要有双向链接就是有关联——知识检索 BFS 不区分链接类型。

**来源**：`[[素材箱中的实际文件名]]`。与相关链接之间空行分隔，相互独立。

## MOC 维护规范

每个 MOC 统一结构：
```
# MOC-主题
> **AI 工作索引**：使用场景说明
正文概述（2-3 句）
## 子分类
- [[卡片名]]
## 相关 MOC
- [[MOC-xx]]
```

**操作后强制自检 8 项**：frontmatter 正确（`source_type: MOC`）→ 标题 `# MOC-主题` → 节头统一 `##` → 死链 0 → 重复 0 → 空节 0 → 无编译临时痕迹 → 相关 MOC 无双前缀。

**新增卡片归入**：找到匹配 MOC 和子分类 → 追加 `- [[卡片名]]` → 跑 8 项自检。

## 编译后维护

1. **同步卡片总数**：由编译收口脚本 `--session-end` 自动同步 `index.md`、`CLAUDE.md`、`01-关于我/人物档案.md`。详见「知识编译」Skill。不再手动维护。
2. 更新 `log.md` 编译记录
3. 新卡片归入至少一个 MOC
4. 调用记录追新卡片行（count=1）
