# 网络诊断模块（数据层体检 · S2.2）

> 版本：v1.1 · 2026-07-08
> v1.0→v1.1：**独立 Skill 降级为体检子模块 + 支持单模块直达**。原「知识网络诊断」Skill 并入数据层体检。两种调用路径：① 体检深度模式 S2.2 加载；② 用户说「网络诊断」时由体检前置路由**直达**本模块（不跑其他步骤，~1-3 min）。脚本迁至 `数据层体检/scripts/analyze_network.py`，分析逻辑不变。

全量分析知识网络的图拓扑结构——卡片是节点，`[[wikilink]]` 是无向边。回答一个核心问题：**这个网络是健康的吗？信息能从任意节点流到任意节点吗？**

**输出模式**：被体检 S2.2 调用时 → 结果写入体检报告的「深度扫描结果·网络拓扑」节。被前置路由直达时 → 输出独立报告到 `00-运转日志/体检报告/YYYY-MM-DD-知识网络诊断.md`。

## 与链接校验模块（S2.1）的分工

| 维度 | 链接校验（S2.1） | 本模块（S2.2） |
|------|--------------|---------|
| 审什么 | 链接**语义**质量（应链未链/链了不相关/密度异常） | 网络**拓扑**健壮吗（连通/路径/瓶颈） |
| 粒度 | 逐对链接语义 | 全局图结构 |
| 互补 | 修语义 = 让遍历时边的关联可信 | 修拓扑 = 让遍历时路径可达 |

## 分析流程

### 第一步：运行拓扑扫描

```bash
python3 "AI协作工具/skills/数据层/数据层体检/scripts/analyze_network.py" \
  --cards-dir "02-知识卡片/知识卡片" \
  --json \
  --output-index "02-知识卡片/.network_index.json" \
  --output-dashboard "02-知识卡片/dashboard_data.json"
```

> ⚠️ `--output-index` 和 `--output-dashboard` 不可省略。下游（体检入度/知识检索枢纽排名/看板）依赖这两个文件。

JSON 输出包含：

| 字段 | 含义 |
|------|------|
| `metrics` | 核心拓扑指标（节点数、边数、连通分量、最短路径、直径、度、关节点、死链） |
| `grade` | 综合健康度：🟢/🟡/🔴 |
| `issues` | 不达标的项及等级 |
| `degree_percentiles` | 度分布 P10-P99 |
| `top_hubs` | Top 20 枢纽节点（按度中心性） |
| `top_bridges` | Top 20 桥梁节点（按介数中心性） |
| `articulation_points` | 关节点清单 |
| `small_components` | 非最大连通分量的卡片清单 |
| `farthest_pair` | 网络中最远的卡片对 |

### 第二步：健康阈值判定

| 指标 | 🟢 健康 | 🟡 警告 | 🔴 严重 |
|------|:-------:|:-------:|:-------:|
| 连通分量数 | = 1 | 2-3 | > 3 |
| 平均最短路径 | ≤ 5 跳 | 5-7 跳 | > 7 跳 |
| 网络直径 | ≤ 8 跳 | 8-12 跳 | > 12 跳 |
| 关节点 | = 0 | 1-3 | > 3 |
| 孤立节点 | = 0 | 1-5 | > 5 |
| 死链（卡片间） | = 0 | 1-10 | > 10 |

> **死链计算**：只统计 `[[link]]` 指向另一张卡片但目标文件不存在的链接。指向素材箱/MOC/01-关于我的跨目录链接不算死链。脚本输出的 `dead_links` 包含跨目录链接（需人工剔除），手动检查时聚焦卡片间死链。

### 第三步：输出诊断结果

诊断结果直接写入本次体检报告的「深度扫描结果 · 网络拓扑」节（不再单独出报告），内容：拓扑指标表、度分布解读、枢纽/桥梁 Top 15、最远卡片对、非最大分量卡片、与上次的指标对比。

**性能基线趋势**：每次运行 `analyze_network.py --output-index` 时自动追加关键指标到 `02-知识卡片/.network_baseline.jsonl`。对比最近 4 次运行：
- 网络直径连续增长（2 次以上）→ 🟡 网络在稀疏化
- 连通分量增加 → 🔴 网络碎片化
- 平均度连续下降 → 🟡 新卡链接质量在下降（触发体检 S4 编译反馈闭环）

### 第四步：修复

**机械修复（自动执行）**：
- 孤立节点（出入度=0）→ 从语义最近的高入度卡片补回链，直接写裸 `[[链接]]`
- 小连通分量（2-5 张卡）→ 找出与主分量语义最相关的卡片建链
- 关节点 → 在关节点连接的两个子域之间加直接链接做冗余

**语义判断（列清单让用户确认）**：
- 该不该合并语义高度重叠的卡片
- 某张卡该不该归入另一个 MOC
- 最远卡片对是否反映真实的知识隔离（故意隔离则无需处理）

**修复后复扫**：修复完成后重新跑脚本，验证指标回到 🟢。

## 铁律

- **全量扫描，不抽检**：千级节点全对最短路径可接受；卡片增长到 5000+ 时改用采样估计
- **脚本做机械分析，AI 做语义判断**：脚本算指标、列清单；AI 读卡片正文判断「该不该补链」「该不该合并」
- **与链接校验模块互补，不重复**：S2.1 审语义 → S2.2 审拓扑 → 体检报告汇总两者
- **修复后必须复扫验证**：复扫确认指标回绿才算完成

## 异常处理

| 场景 | 触发条件 | 处理动作 |
|------|---------|---------|
| 卡片目录为空 | `02-知识卡片/知识卡片/` 无文件 | 告知用户知识库未初始化，终止分析 |
| 连通分量 > 1 | 网络出现裂痕 | 🔴 列各分量卡片清单，对非主分量逐张补链 |
| 关节点 > 0 | 存在单点瓶颈 | 🟡 列关节点清单，给桥接路径补冗余链接 |
| 平均路径 > 7 | 网络过于稀疏 | 🟡 扫低度卡片（P25 以下）补互见链接 |
| 脚本执行超时 | 卡片数过大（>5000） | 降级为采样估计：随机 500 节点子图计算，标注「采样估计」 |
| 死链 > 0（卡片间） | 重命名/删除后链接未更新 | 🔴 列死链清单，同日修复 |
