# 00-运转日志·子宪法

> 版本：v1.25 · 遵从根宪法 v2.0 · 2026-06-15

## 目录结构

```
00-运转日志/
├── 对话记录/                      ← 每场对话存档（按日归并，6点为界）
├── 日报/                          ← 每日跨日快照
├── 周复盘/                        ← 每周轻量复盘
├── 体检报告/                      ← 知识库健康检查报告
├── 框架审核/                      ← 框架层审核全量报告 + token审计报告（框架层审核 Skill 产出）
├── 知识卡片调用记录.md            ← 全库卡片调用次数表（格式：| [[卡片名]] | 调用次数 |，新卡 count=1，与全部卡片一一对应）
└── CLAUDE.md
```

## 各子目录约定

- **对话记录**：按「对话存档」Skill 流程管理。追加/整理修订规则见 Skill 定义
- **体检报告**：知识库维护报告统一放这里。不散落在 AI协作工具或其他地方
- **框架审核**：框架层审核产出（框架层审核 Skill）
- **日报/周复盘**：分别由对应 Skill 维护

## 对话存档·格式与铁律

遵从「对话存档」Skill 的完整定义，此处仅保留格式速查。

### 文件命名

`[日期]-[主题关键词].md`。「日」以早上 6 点为界。

### 密度铁律

- 要点化默认：先想「能不能 3-5 个 bullet 讲完」
- 单日超 400 行 → 二次浓缩（合并相似主题、删已被知识库承接的认知、过程来回归并为结论）

### 不存档

纯工具调用确认、无新认知的闲聊、重复性操作。

### 信息闭环

有价值知识/认知 → `02-知识卡片/` 或 `01-关于我/`，不在存档中重复沉淀。

## 铁律

- 不在此目录存放不属于运转日志的文件（如系统优化方案→归根目录 CHANGELOG，工具清单→归 AI协作工具/skills/工具管理/）
- 新增子目录后更新本文件的目录结构节
