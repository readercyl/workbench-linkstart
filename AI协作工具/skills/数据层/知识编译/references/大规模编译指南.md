# 大规模编译指南

> 知识编译 Skill 参考文件——素材箱 `_待处理/` 下累积 ≥10 篇素材时查阅。

## 分批策略

每批控制在 **≤15 张新卡**，不论来自几篇素材。

| 素材密度 | 每批篇数 | 典型场景 |
|---------|---------|---------|
| 低密度（每篇 1-3 张） | 3-5 篇 | 视频日记、短文章、简单笔记 |
| 中密度（每篇 3-8 张） | 2-4 篇 | 公众号文章、课程笔记 |
| 高密度（每篇 8-20+ 张） | **1 篇即一批** | 长文深度拆解、学习会逐字稿 |

**高密度单篇处理**：一篇产出 ≥15 张卡 → 编译完这篇后立即 `--force` 收口，不和其他素材混合。

## Agent 并行约束

- 3-5 篇可并行委派给 Agent，每 Agent 一个素材
- ≥10 篇：分批并行，每批 ≤5 个 Agent
- Agent 提示词必须嵌入卡片格式规格 + 写卡自检清单 + 对抗式查漏+过滤要求（含条件②决策可调用 + 条件③信息有价值，见 SKILL.md 步骤 3）
- Agent 只写卡片文件到 `02-知识卡片/知识卡片/` + 追加队列文件。禁止写 log.md / index.md / MOC / 调用记录 / 人物档案
- 写卡前 grep 关键词扫已有卡片。≥80% 语义重叠→跳过；50-80%→写入+链接到已有卡
