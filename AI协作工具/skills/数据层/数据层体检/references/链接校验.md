# 链接校验模块（数据层体检 · S2.1）

> 版本：v2.2 · 2026-07-08
> v2.1→v2.2：**独立 Skill 降级为体检子模块 + 支持单模块直达**。原「知识链接校验」Skill 并入数据层体检。两种调用路径：① 数据层体检深度模式 S2.1 加载；② 用户说「链接审计」时由体检前置路由**直达**本模块（不跑 S1/S3/S4/S5，~2-5 min）。审计逻辑不变。
> v2.0→v2.1：移除「链接过多」判定。

全库审计链接的语义质量。知识库卡片 `**相关链接**` 节只写裸 `- [[链接]]`，本模块从语义维度审计链接质量（拓扑维度见同目录 `网络诊断.md`）。

**输出模式**：被体检 S2.1 调用时 → 结果写入体检报告的「深度扫描结果·链接审计」节。被前置路由直达时 → 输出独立报告到 `00-运转日志/体检报告/YYYY-MM-DD-链接审计报告.md`。

## 审计流程

### 第一步：链接密度扫描（机械）

```bash
python3 "AI协作工具/skills/数据层/数据层体检/scripts/analyze_network.py" \
  --cards-dir "02-知识卡片/知识卡片" \
  --json --low-link-threshold 3
```

检查 JSON 输出中 `low_link_cards` 字段。

扫描全库每张卡片的 `**相关链接**` 节，统计链接数量：

| 维度 | 判定标准 | 动作 |
|------|---------|------|
| 链接过少 | < 3 条 | 标注 ⚠️，提示可能为孤立卡片 |
| 正常 | ≥ 3 条 | 跳过 |

链接数量不设上限。枢纽卡天然高链接——只关心语义质量：链了的是不是真相关。

### 第二步：语义相关性审计（链了但不相关）

逐对审计现有链接的语义相关性——A 链了 B，但 A 和 B 的内容真的构成关联吗？

**预筛**：先跳过以下自动通过的情况，减少人工审计量：
- 两张卡 `**来源**` 指向同一素材文件 → 自动通过
- 两张卡在同一 MOC 中（通过目录路径推断）→ 自动通过
- 两张卡标题有显著共享术语（bigram 重叠 > 0）→ 自动通过

**审计方法**：剩余链接对分批（每批 ≤20 对），使用 Agent 工具并行分发。每对读取双方卡片正文全文，逐对判断：
- ✅ 语义相关：两张卡讨论同一领域/话题/方法体系
- ❌ 不相关：两张卡话题完全独立，链接无依据
- ⚠️ 弱相关：有关联但很牵强（如仅共享一个通用词）

### 第三步：应链未链检测（语义相近但未互链）

检测语义高度相近但 `**相关链接**` 中未互相引用的卡片对。

**启发式候选生成**（机械预筛，不做全库 NxN 语义比对）：
- 同一 `**来源**` 素材编译出的卡片，未互链 → 标记
- 标题 bigram 重叠率 > 50%，未互链 → 标记
- 共享 >= 2 个相同的出链目标（共同引用），未互链 → 标记

**审计方法**：候选对分批（每批 <= 20 对），Agent 并行判断是否为「应链未链」。读取双方全文，确认语义相关后建议补链。

### 第四步：输出审计结果

审计结果直接写入本次体检报告的「深度扫描结果 · 链接审计」节（不再单独出报告），结构：

```markdown
### 链接审计（S2.1）

#### 链接密度
- 全库卡片：N
- 链接过少（<3）：X 张（清单）
- 正常范围：Z 张

#### 语义相关性（链了但不相关）
- 总链接对：N · 自动通过：X 对 · 需审计：Y 对
  - ✅ 相关：A 对 · ❌ 不相关：B 对（建议移除）· ⚠️ 弱相关：C 对（待用户确认）
- 明细

#### 应链未链
- 候选对：N（建议补链：X 对 · 不需要：Y 对）
- 明细

#### 修复清单
| 文件 | 操作 | 详情 | 状态 |
|------|------|------|------|
```

## 修复

- 链接密度异常 → 列清单让用户确认后处理
- 语义不相关 → 列候选让用户确认后移除链接
- 应链未链 → 列候选让用户确认后补链
- 不改卡片正文内容，只改 `**相关链接**` 节

## 铁律

- **密度全量，语义抽样**：密度扫描全量机械执行（脚本即可）。语义审计只针对预筛出的候选对，不做全库 NxN 语义比对
- **逐对读取全文**：判断语义相关性必须读取双方卡片正文，只看标题无法判断
- **不改正文**：只动 `**相关链接**` 节，不动卡片正文内容
- **先预筛再审计**：机械预筛产出候选对，AI 只审候选。不把手动审计当预筛
- **确认后执行**：修复清单必须经用户确认后批量执行，不自动改卡

## 异常处理

| 场景 | 触发条件 | 处理动作 |
|------|---------|---------|
| 密度扫描无异常 | 全库链接数均 ≥3 | 标注 🟢，跳过密度相关修复 |
| 语义审计候选为空 | 预筛后无可疑链接对 | 正常——说明链接质量高，记录「预筛 0 可疑」 |
| 应链未链候选过多 | 候选对 > 200 对 | 按重叠率降序排列，优先审 Top 200 |
| Agent 并行审计失败 | 单个 Agent 超时/报错 | 该对标记「审计失败」，手动补审 |
| 全库链接总数异常 | 总链接数 < 卡片数 x 0.5 | 标注 🔴，可能大量卡片缺相关链接节 |
