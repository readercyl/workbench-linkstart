# 写作模板

> 对话深度整理 Skill · 第二遍写作输出模板 · v3.1

## 写作姿态

深度整理稿的终极检验不是「信息有没有漏」，而是「读者读完能不能记住三个东西、想不想转发给谁」。写作者的角色是**讲故事的人**，不是信息整理师。杂志特稿的写法——有故事弧线、有情绪节拍、有呼吸感——比参考文档的写法更能实现这个目标。

**三要**：
- 开篇第一句要带读者进入场景，不是做评价
- 要保留对话中的「灵感爆发时刻」和情绪高点，它们是读者的记忆锚点
- 段落要有呼吸感——短段落制造节奏，关键判断单独成行

**三不要**：
- 不要用 `---` 硬分隔板块——用叙述过渡让板块之间自然流动
- 不要把每个说话人的每句话都公平对待——深度优先，允许次要信息列表化
- 不要把情绪当噪音过滤掉——「我靠！宝藏就在我自己这里！」比方法论步骤更容易被记住

## 各部分写作要求

### 标题

简洁有力，让读者一眼知道这篇在讲什么。格式：`[分享人/活动名]：[核心主题]·深度整理`

**示例**：
- `杨涛：AI 创业赚钱方向与实战答疑·深度整理`
- `传术师大会 2026·深度知识整理`
- `刘小排：普通人如何借助 AI 编程成为超人·深度整理`

### 开头段

开头两段必须做两件事：
1. 让读者一秒判断「这是不是我关心的话题」
2. 给出一个明确的价值判断，让读者觉得「这篇有东西，值得读完」

**不要铺垫、不要背景介绍**——直接给判断。

**示例**（好的开头）：
> 刘小排用一句话重新定义了 AI 编程：**写流程 + 拼积木 + 管员工**。三者都不是传统意义上的「编程」。
>
> 这篇整理来自他在超级 AI 大航海的直播分享，核心回答一个问题：零编程基础的人，怎么借助 AI 把模糊想法变成产品。

**反例**（不好的开头）：
> 2026 年 5 月 14 日，超级 AI 大航海举办了第 4 场直播分享。本次分享邀请到了刘小排老师，他是 AI 编程教育者，生财有术圈友。刘老师围绕 AI 编程这个话题进行了深入讲解...（铺垫太多，没给判断）

### 板块总览

紧接开头，列出全文板块清单。每个板块一行，包含板块名和一句话提要。

**格式**：
```
**本次共 5 大板块**：
一、AI 编程的本质 —— 不是写代码，是写流程、拼积木、管员工
二、SPEC 驱动法 —— 像苏格拉底一样梳理需求
三、唯一门槛 —— 勇气 + 耐心，慢就是快
四、工具与实操 —— 推荐工具和关键技巧
五、金句汇总
```

### 核心板块正文

每个核心板块包含以下要素：

**子主题叙述**：
- 用第三人称叙述，不要「说话人 X 说」
- 原话用 `>` 引用格式
- 操作步骤用数字列表或分步说明
- 技术数据用表格

**原话引用规范**：
```
> "原文中的判断句。"
```
- 原话必须与原文逐字一致，不得改写
- **不署名**：引文不标注说话人和文件来源——知识整理的重点是内容，不是谁说的。开头段交代参与人和背景即可
- 引用后可以加 AI 的解读，但必须标注「AI 注：」
- 不要为了「忠于原文」保留无信息量的句子

**子主题之间用 `### N.N 子主题名` 分隔**。

### 边界

每个核心板块末尾必须包含 `### 边界` 节。

边界的写法：不是敷衍一句「视情况而定」，而是具体说明什么场景下这个方法不适用。

**示例**（好的边界）：
```
### 边界

- SPEC 驱动的苏格拉底式追问最适合「需求模糊但目标明确」的场景。如果你的需求本身已经非常精确（比如「修复这个 API 的返回格式」），追问反而浪费时间
- 语音输入虽然高效，但不适合包含大量技术术语或精确参数的需求——口述容易出错
- 这个方法对 AI 编程工具有效，但迁移到其他领域（比如让 AI 写文章）时，追问的颗粒度需要调整
```

**反例**（不好的边界）：
```
### 边界
具体情况具体分析，需要根据实际场景判断。
```

### 金句汇总

文末按主题分类陈列金句。每条金句：
- 必须是原文原话，不可改写
- 以 💡 开头
- 按主题分节（如「关于 AI 编程」「关于创业」「关于个人成长」）

**格式**：
```
## N、金句汇总

### 关于 [主题A]
💡 "原话一"

💡 "原话二"

### 关于 [主题B]
💡 "原话三"
```

## 自检清单（内部使用，不写入产出文件）

写作完成后，写作 Agent 必须逐项自检（评审 Agent 也会对照此清单审查）：

- [ ] **开头两段能让人判断话题 + 感到有东西**——这是最重要的
- [ ] 开篇有板块总览（「本次共 N 大板块」）
- [ ] 最有价值的洞察排第一、给最大篇幅
- [ ] 没有「说话人 X」残留和口语化碎词（「然后」「那个」「就是」「对吧」）
- [ ] 人物有身份标识（真实名或角色特征），不是编号
- [ ] 每个核心板块有 `### 边界`
- [ ] 技术数据用表格或精确数字，不用「大概」「差不多」
- [ ] 文末有金句汇总区，金句是原话（💡 标记）
- [ ] AI 的判断和原文表述可区分（标注「AI 注：」）
- [ ] 读者不需要听过原始录音就能完全理解
- [ ] 读者读完能说出「这篇最有价值的三个信息点是什么」
- [ ] 情绪 4-5 的标注段落在整理稿中有充分展开（不是一句话带过）
- [ ] 全文有至少 2-3 个情绪高点，且不集中在开头
- [ ] 对话弧段「高潮」的段落被放在板块核心位置、给最大篇幅

## 基于标注索引的叙事编排指南

写作 Agent 从标注索引中拿到的不只是「谁说了什么」——你同时拿到了情绪强度（1-5）和对话弧段（铺垫/高潮/转折/收束）。这些字段是叙事编排的量化依据。

### 情绪强度的使用

| 情绪分值 | 写作处理 |
|---------|---------|
| 5 | **全文保留，单独成段**。这是读者的记忆锚点——原话引用 + 上下文交代 + AI 注解。给它最大篇幅 |
| 4 | **重点叙述**。保留原话引用，还原对话张力和情绪色彩。不要压缩成客观陈述 |
| 3 | **观点提炼 + 简要引用**。摘核心判断，保留说话人口吻但可以精简 |
| 2 | **归入背景或合并叙述**。多人对同一话题的正常讨论，可合并成一段综合叙述 |
| 1 | **可以压缩或略过**。平铺直叙的技术说明/操作步骤，提取有用信息但不展开叙述 |

**铁律**：情绪 4-5 的标注段落不得被压缩成一句话带过。它们是读者读完能记住的东西。

### 对话弧段的使用

| 弧段 | 写作处理 |
|------|---------|
| 铺垫 | 放在主题板块开头，一句话建立上下文即可。不需要大段背景叙述 |
| 高潮 | **板块的核心段落**。情绪 4-5 + 弧段「高潮」= 这个板块最重要的内容。给它最大篇幅、最完整的叙述 |
| 转折 | **承上启下的张力点**。放在铺垫和高潮之间，或两个子主题之间。用转折句引出新观点：「但/然而/反过来说」 |
| 收束 | 放在板块末尾，或作为自然段落的收尾句。收束标注往往自带总结性判断——可以直接用作板块的结论句 |

**同板块内的弧段顺序**：铺垫（建立上下文）→ 转折（引入张力）→ 高潮（核心内容）→ 收束（结论）。不是每个板块都有四个弧段——小板块可能只有高潮+收束。

### 全文情绪曲线

写完初稿后，对照标注索引的情绪分值，回看整理稿的情绪节奏：

1. 全文有没有至少 2-3 个情绪高点（4-5 分区）？→ 没有的话，标注索引里的高点被压缩了——回去补
2. 情绪高点是否集中在开头 1/3？→ 好的叙事弧是「逐步升温」，不是「开头即巅峰后一路下滑」
3. 板块之间是否有情绪起伏？→ 连续两个板块都是情绪 1-2 → 检查是否该合并或删掉一个

## 写作原则

**有用优先，形式不限。** 不纠结是方法论还是知识体系还是判断还是观点——形式不重要，信息点有没有用才重要。能给读者带来新知、新观点、辅助判断的依据、可操作的方法——其中任何一个，这篇文章就有价值。

**精炼优先，字数不限。** 不以字数设限。可以 3000 字，也可以 15000 字——取决于内容有多少有价值的信息点。语言精炼，以表达清楚意思为标准，不凑字、不灌水。

**亲历者复盘，不是百科全书编辑。** 淘金——大量沙子扔掉，只留金粒。只留下最值得留下的，给核心洞察最大篇幅。

**共享是最终检验。** 一篇文章的终极价值判断是：读者读完会不会觉得「这东西有用，得转给 XX」。触发转发的四个要素：新知、新观点、辅助判断、可操方法。

**来源节单向引用**：`**来源**` 是知识卡片的专属字段，用于卡片→源材料的单向引用。源材料文件（含深度整理稿）禁止嵌入 `[[卡片名]]`、禁止添加「已编译卡片」等反向引用列表。Obsidian 自动处理反向链接。
