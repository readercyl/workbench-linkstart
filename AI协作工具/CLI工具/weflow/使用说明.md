# WeFlow 微信记录清洗 · 使用说明

> 定位：微信 WeFlow 导出聊天记录 → 双层清洗 → 高价值信息提取
> 协议：Python 脚本 + 大模型语义过滤
> 触发：用户说「清洗 WeFlow 记录」/ 素材箱出现新 .txt 文件

## 管道架构

```
WeFlow 导出 .txt → Layer 1 结构清洗 → Layer 2 语义过滤 → 清洗后 .txt + .bak
```

### Layer 1：结构清洗（Python）
- 翻译微信 ID → 真实姓名
- 去除系统噪音（时间戳、系统消息、红包提示等）
- 原地重写，保留 .bak 备份

### Layer 2：语义过滤（DeepSeek v4-flash）
- 高精去噪
- 保留：教练指导、技术问答、常青链接
- 排除：闲聊、表情包、重复消息、纯表情

## 脚本说明

| 文件 | 用途 |
|------|------|
| `weflow_filter_pipeline.py` | 总管道脚本 |
| `weflow_processor.py` | 微信记录处理器 |
| `weflow_semantic_filter.py` | 语义过滤脚本 |
| `weflow_semantic_filter_SOP.md` | 语义过滤 SOP |
| `run_batch_clean.py` | 批量清洗入口 |

## 使用方式

由「WeFlow记录处理」Skill 自动编排，AI 不单独调用脚本。Skill 串联 Layer 1 + Layer 2，执行完毕后通知用户。

## 安全规则

- 原文件始终保留 .bak 备份
- 不删除原始 WeFlow 导出文件
- 清洗后文件原地覆盖，不是新建
