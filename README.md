## 这是什么

这是**人 + AI + 知识库三者协作**的工作台分享版。仓库保留协作协议、知识编译、对话存档、日报、巡检和框架审核等完整流程；Hook / Cron 需要按下方初始化步骤配置后才会运行。

**核心思想**：知识库不是用来存东西的，是用来被 AI 调用的。卡片写成那样、链接打成那样、目录结构那样——全是为了让 AI 能读懂、检索、推导。

## 三步启动

### 方式一：让 AI 带你走

把这段发给你的 AI（Claude Code / Codex）：

```
我有一个工作台分享库，请帮我搭建个人知识库。
仓库地址：https://github.com/readercyl/workbench-linkstart
请先克隆到本地，然后根据 CLAUDE.md 引导我完成初始化。
```

支持读取仓库和编辑文件的 AI 可以：克隆仓库 → 读 CLAUDE.md → 和你对话填人物档案 → 教你怎么写第一张卡片。

### 方式二：自己研究

```bash
git clone https://github.com/readercyl/workbench-linkstart.git
cd workbench-linkstart
# 从 README.md 和 CLAUDE.md 开始读
```

### 方式三：给你的 AI 一份说明书

把仓库链接直接发给你的 AI，说：「读一下这个仓库的 CLAUDE.md，以后用这套协议跟我协作。」

### 启用本地自动化（可选）

仓库不会默认执行任何脚本。需要 Hook 时，把示例配置复制为本机配置：

```bash
mkdir -p .claude
cp .claude/settings.example.json .claude/settings.json
cp .env.example .env
```

然后只在本机 `.env` 中填写需要的 API Key、GetNote topic 等配置；`.env` 已被 Git 忽略，禁止提交。
Hook 会自动读取根目录 `.env`。手动运行其他 API 脚本前，请先把对应变量导入当前终端环境。

## 系统架构

```
workbench-linkstart/
├── CLAUDE.md              ← AI 入口协议（灵魂文件）
├── README.md              ← 你在看的这个
├── index.html             ← 可视化介绍页
├── .env.example           ← 本机环境变量模板（无真实值）
├── .claude/               ← Hook 配置示例
├── 01-关于我/             ← 你的个人说明书
├── 02-知识卡片/           ← 原子卡片 + MOC 索引
├── 00-运转日志/           ← 对话存档 / 日报 / 周复盘
├── 03-认知工坊/           ← 外部 Skill（调研、学习、讨论）
├── 04-内容生产/           ← 内容创作工具和方法论
├── 05-项目运营/           ← 项目管理
├── 06-对外分享/           ← 对外输出
├── 素材箱/                ← 待编译的原始素材
└── AI协作工具/            ← Skill + Hook + 全系统巡检
    ├── skills/
    │   ├── 运营层/         ← 存档·日报·周复盘·巡检
    │   ├── 数据层/         ← 编译·检索·体检·深度整理
    │   └── 框架层/         ← 审核·工具管理·自修正
    └── hooks/              ← 自动化脚本
```

## 维护系统

工作台自带三层维护系统，确保知识库不会腐化：

| 层 | 负责 | 我怎么触发 |
|----|------|-----------|
| 🟢 运营层 | 每日存档、日报生成、巡检 | 配置 Hook 后自动 + 手动触发 |
| 🟡 数据层 | 知识编译、深度检索、全库体检 | 需要时手动触发 |
| 🔴 框架层 | 全系统审核、工具管理、自修正 | 结构性改动后按协议触发 |

## 知识卡片格式

每张卡片四个区：

```markdown
---
created: 2026-07-14
source_type: 文章
status: seed
---

# 标题即核心论点

正文。[[双向链接]] 自然嵌入句子中。

**相关链接**：
- [[卡片A]]
- [[卡片B]]

**来源**：[[原始素材文件名]]
```

格式规格详见 `02-知识卡片/CLAUDE.md`。示例卡片在 `02-知识卡片/知识卡片/`。
