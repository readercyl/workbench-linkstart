# 工作台方法论

> 这套系统的核心思想。推荐给愿意深读的人。

## 为什么需要工作台

大多数人用 AI 的方式是「每次对话从零开始」——AI 不知道你是谁、在做什么、之前讨论过什么。每次都要重新交代背景。

工作台解决的就是这个问题：**让 AI 有记忆、有上下文、有协作流程**。

具体做法：
1. **人物档案**——告诉 AI 你是谁
2. **知识卡片**——把你的认知写成 AI 能检索的原子单元
3. **双向链接**——卡片之间的连线形成知识网络，AI 沿着链接推导
4. **对话存档**——每场有实质内容的对话自动归档
5. **维护系统**——自动巡检，防止知识库腐化

## 核心设计理念

### 1. 宪法体系

不是给 AI 一本厚厚的说明书，而是**分层协议**——根宪法管全局原则，子宪法管具体模块规则。AI 进入不同模块自动切换行为模式。

### 2. 知识编译

外部知识（文章、视频、对话）不会自动变成可检索的卡片。编译流程：去噪 → 提炼核心论点 → 建立双向链接 → 归入 MOC。

### 3. 维护系统

三层：
- **运营层**（每天跑）：存档、日报、巡检
- **数据层**（按需跑）：编译、检索、体检
- **框架层**（改系统时跑）：审核、工具管理、自修正

知识库不维护就会腐化——孤立卡片、死链、重复内容。这套维护系统是让它持续可用的保障。

### 4. Skill 即 SOP

每个 Skill 是一份「AI 看了就知道怎么执行」的 SOP 文档 + 配套脚本。不是黑盒代码，是透明的工作流描述。

### 5. 全系统一致性

系统的每个组件都引用同一套宪法——改一处，所有关联文件同步更新。框架层审核确保不会「改了 A 忘了 B」。

## 适用场景

| 场景 | 工作台怎么用 |
|------|-------------|
| 碎片化信息管理 | flomo/笔记 → 素材箱 → 编译为原子卡片 |
| AI 协作效率 | AI 读人物档案 + 知识库，每次对话都有上下文 |
| 个人成长追踪 | 对话存档 + 日报 + 周复盘，看到自己的思维演变 |
| 知识体系构建 | 卡片 → 双向链接 → MOC → 知识网络自生长 |
| 内容创作支撑 | 检索知识库 → 找到关联卡片 → 形成文章骨架 |

## 局限与方向

**优势**：知识管理流程完整——编译、检索、体检、存档的链路均有明确协议和工具支撑，分享版附带一组可直接检查的连通示例卡片。

**待完善**：内容生产模块。框架已搭，实际创作工具和方法论还需打磨。

**进化方向**：
1. 内容生产工具链完善
2. 多 AI 工具兼容性增强
3. 团队协作场景支持
4. 知识卡片自动发芽（跨领域关联发现）
