# hooks·自述

> 本目录存放自动化钩子脚本。`.sh` 文件在 Obsidian 中不可见——这里是它们的说明。

## 当前脚本

### `cron-refresh.sh`

每次启动时读取 `.claude/scheduled_tasks.json`，刷新其中全部 `recurring: true` 任务的 `lastFiredAt`，防止 7 天过期上限导致任务静默消失。脚本不保存个人任务 ID，带锁保护，与 cron daemon 并发写安全。

### `workbench-prep.sh`

工作台准备——AI 按顺序执行各项准备任务。脚本自动根据自身位置识别工作台根目录，也可用 `WORKBENCH_ROOT` 覆盖。受双重闸门限制：

| 闸门 | 位置 | 规则 |
|------|------|------|
| 时间窗口 | 脚本内 | 6:00 ≤ 当前小时 < 18:00，否则静默退出 |
| 消息匹配 | AI 判断 | 脚本产物注入后，AI 检查用户是否发送了早上问候语（如「早上好」「早啊」「早」等），不是则跳过全部 |

负责 5 项每日自检：周一提醒 / 日报（整理+生成） / 视频日记导出 / 知识编译 / 安全网（深度内容候选）。18:00 后不执行，推迟到次日。视频日记检查只有在本机配置 `GETNOTE_TOPIC_ID` 时才启用。

**输出格式**：闸门 2 一行 + 顺序执行标记 + 待执行任务清单（每项一行，不展开）。AI 执行时从对应 Skill 读取详细步骤。

**修改方式**：终端直接编辑 `.sh` 文件。改完后 AI 协作工具子宪法的「Hook 脚本改动后必须测试」规则生效。

## 启用方式

仓库默认不执行 Hook。复制根目录 `.claude/settings.example.json` 为 `.claude/settings.json` 后启用；真实 Key 与个人 ID 只放本机环境变量或 `.env`，不得写回仓库。
