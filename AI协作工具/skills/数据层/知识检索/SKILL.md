# 知识检索

> 版本：v3.2 · 2026-07-14 · 调用卡片数量分层上限

> ⚠️ 最高优先级·前置强制 Skill。AI 每次回答前必须执行以下自检，不自检就直接回答 = 违反根宪法。

## 触发条件

**回答前先自问：这个回答需要依据知识 Wiki（`02-知识卡片/`）中的信息吗？** 需要→调库。不需要→跳过。

### 触发（需要调库）

| 场景 | 关键词 |
|------|--------|
| 涉及判断/决策/方向选择 | 「怎么选」「该不该」「先做什么」 |
| 答疑/复盘/总结 | 「为什么」「复盘一下」「之前怎么做的」 |
| 内容创作/选题 | 「写一篇」「做什么选题」「怎么表达」 |
| 心智调试/自我认知 | 「我为什么总是」「感觉自己」「怎么调整」 |
| 产品/商业/项目讨论 | 「怎么做产品」「定价」「这个生意」 |

### 不触发（不需要调库）

| 场景 | 示例 |
|------|------|
| 纯外部调研 | 搜工具、查价格、找 GitHub 项目、技术配置 |
| 纯系统维护 | 改配置、修 bug、调参数、文件操作 |
| 纯工具执行确认 | 「帮我发个消息」「查一下日程」「打开文档」 |
| 纯闲聊 | 日常寒暄、无关判断的随意对话 |

**交叉场景的判断标准**：如果用户的提问同时涉及外部信息和内部知识——比如「调研了 X 工具，结合我之前的情况选哪个」——后半句「之前的情况」触发调库，只调那部分的卡，不因有外部调研成分就全跳过。

## 角色

维护系统数据层——知识库检索引擎。MOC定位→BFS遍历→语义筛选→带引用回答。调用是默认状态不特殊，不向用户汇报调用过程。

## 调用流程

### 第一步：识别任务类型 → 定位 MOC

| 任务 | MOC |
|------|-----|
| 写内容/做 IP/选题 | `MOC-内容创作能力` |
| 卖货/谈单/做发售 | `MOC-销售与转化` |
| 产品定价/服务设计 | `MOC-产品与服务` |
| 选方向/评估机会 | `MOC-战略思维与创业` |
| 拖延/卡住/抗内耗 | `MOC-执行力与时间` |
| 心态/自我认知/情绪 | `MOC-认知与心智` |
| 工具/AI/工作流 | `MOC-AI协作方法论` |
| 知识库本身 | `MOC-知识管理元层` |
| 商业诊断/定价/盈利 | `MOC-商业模式诊断方法论` |
| 对标分析/模仿/拆解 | `MOC-战略思维与创业` |

跨场景任务涉及多个 MOC。

### 第二步：浏览 MOC → 定位候选卡片

1. 打开第一步定位的 MOC 文件
2. 在对应节标题下找到候选卡片名——卡片名本身就是标题，通过标题判断是否相关
3. **数量指引**：小型 MOC 直接读取全部条目；大型 MOC 单路初筛 **8-15 张**候选卡，多 MOC 并行时每路初筛 **5-8 张**，合并后去重
4. 标题不够明确时 Read 卡片全文

### 第三步：读卡 → 调用（三路径混合）

三种路径叠加使用，不是三选一。AI 内部完成，不对外输出。

**路径 1 — 网络深度遍历（主路径）**：从候选卡片出发，沿所有双向链接做 BFS 遍历。typed link 类型前缀已全局取消（v1.34），所有链接现在都是裸 `[[链接]]`，BFS 不区分边的类型——每条链接都可能通向相关卡片。遍历深度自适应：

- 先走 **2 跳**，收集沿途卡片，检查跨 MOC 覆盖度：Top 20 已全部来自同 1 个 MOC 且问题领域集中 → 停在 2 跳；跨 MOC 卡片 < 3 张且问题明显跨域 → **扩到 3 跳**
- 遍历深度依据最近一次数据层体检报告调整；没有体检数据时默认 2 跳，结果不足再扩至 3 跳

**检索深度自声明（硬门控 · 可复核证据）**：每次知识检索完成后，AI 必须在回答末尾显式输出：`🔍 检索：{N}张种子 → BFS {X}跳 → 收集{M}张 → 语义筛选Top{K}张 → 实际调用{L}张（上限{上限}）`。跨领域问题须声明 MOC 入口数。实际调用数必须 ≤ 分层上限。未输出此行 → 检索未完成，不得给出基于知识库的判断。

**语义筛选规则**（按优先级排序）：
1. 标题与问题的语义相关性（主规则）
2. **含 `<!-- 跨MOC -->` 标注的卡优先**（编译时已标记为跨域桥梁，直接读，不必等 BFS 多路命中推断）
3. **网络索引 Top 15 枢纽卡优先**（读 `02-知识卡片/.network_index.json` 中 `top_hubs` 列表——运行时实时排名，比静态「入度≥3」更准）
4. **status=evergreen 的卡优先**（经过调用验证的成熟卡片）
5. **同时出现在多 MOC 路径中的卡优先**（BFS 多路命中 = 强信号）
6. 取 Top 10-20 张读全文

**调用卡片数量分层上限**：语义筛选后进入回答的卡片数量必须受控。不是取越多越好——卡片越多上下文越重，AI 反而在噪音里丢失判断力。

| 问题复杂度 | 上限 | 说明 |
|-----------|:---:|------|
| 简单（查一个概念/一条判断） | ≤2 张 | 如「什么是 XX」——一张定义卡足够 |
| 中等（需要多角度交叉验证） | ≤5 张 | 如「A 和 B 怎么选」——2-3 个维度交叉 |
| 复杂（跨领域战略问题） | ≤7 张 | 如「当前阶段应该优先做什么」——最多 7 张，多了说明问题该拆 |

**超限处理**：语义筛选后卡片 > 上限 → 不是硬塞更多卡，是**压缩筛选标准**——只留最核心的、删掉边缘相关的、把同类合并。如果压缩后仍然超限 → 说明这个问题本身太大，AI 应主动说「这个问题涉及 X 个领域，我先回答最核心的部分，其他部分可以继续聊」而非硬着头皮塞 10 张卡进去。

**检索声明同步调整**：`🔍 检索：{N}张种子 → BFS {X}跳 → 收集{M}张 → 语义筛选Top{K}张 → 实际调用≤{上限}张`。实际调用数必须 ≤ 上限。

**路径 2 — 多 MOC 并行入口（广度补充）**：跨领域问题同时从 2-3 个相关 MOC 出发，各自执行路径 1 的深度遍历，合并去重后做语义筛选。单领域问题仍走单个 MOC。MOC 条目是裸卡片名无描述，匹配依赖标题质量。不确定时直接 Read 卡片正文。

**路径 3 — 全文 grep（兜底）**：路径 1+2 都召不动时，用 grep 关键词搜 `02-知识卡片/知识卡片/` 正文。专门处理引语、人名、专有名词等字面表述。

### 第四步：直接产出回答

基于扫描结果直接生成回答。可以自然嵌入 `[[卡片]]` 链接方便点开校验。不要写「我调用了 X 卡片」。

### 调用记录

调用记录由对话存档 Skill 每次存档时更新（数据源为存档的「核心引用卡片」字段）。本 Skill 只需在回答中自然使用卡片，不用显式写 `[[wikilink]]`，不用记录调用了什么——存档时 AI 摘要会捕获判断依据。分享版的 11 张示例卡片已全部接入 MOC，卡片网络单连通且没有失效链接；扩充知识库后仍需定期运行数据层体检。

## 卡片知识自下而上的路由

**自上而下（AI 主导）**：MOC → 卡片名 → Read 全文 → 获得判断依据。日常调用路径，AI 后台完成。

**自下而上（用户偶查）**：用户偶尔打开某张卡片，通过双向链接跳转。这依赖网络互联质量——每张卡入度 ≥ 1。

## 反例清单

| # | 反模式 | 替代做法 |
|---|--------|---------|
| 1 | 用户问「怎么优化 A」→ AI 顺手改了 B、C、D | 只看 A，不扩散 |
| 2 | 用户问纯外部调研→ AI 还是扫描了 MOC | 对照不触发清单，跳过 |
| 3 | 调了库但在回答里写「我调用了 X 卡片」 | 结果给用户，过程透明不汇报 |
| 4 | 问题跨多个领域→只看了第一个 MOC | 跨场景任务涉及多个 MOC |

| 6 | 大型 MOC 候选种子只选 3-5 张，导致召回范围过窄 | 单 MOC 初筛 8-15 张，多 MOC 每路 5-8 张；小型 MOC 读取全部 |

## 自我进化

用户纠正检索遗漏/方向偏差 → 走工具自修正流程。

## 铁律

- **回答前默认扫卡**：AI 的内部肌肉记忆。触发条件清单每问必过
- **执行留证据，结果不冗余**：用户看到带引用的结果 + 检索深度声明。检索声明是证据，不冗余展开
- **按名定位，按文判断**：MOC 卡名定位 → Read 全文确认 → 支撑+互见边 BFS 2-3 跳 → 语义筛选 Top 10-20 → 读全文
- **网络互联是调用基础**：卡片入度 ≥ 1、双向链接、每卡归入 MOC——这些不是编译时的形式要求，是调用能否成功的前置条件
- **不是每次回答都需要调库**：对照触发/不触发清单。纯外部调研、系统维护、闲聊——跳过比强行调库更正确
- **调用卡片有上限**：简单 ≤2 张、中等 ≤5 张、复杂 ≤7 张。超限不是硬塞——是压缩筛选或拆问题。卡片多了 AI 反而在噪音里丢失判断力

> **偶尔暴露引用链**：当 AI 的判断显著依赖某张卡片观点时，在回复结尾加一行脚注 `> * 主要依据：[[某卡片]]`。大约每 3-5 次判断出现一次，让知识库从「运转层后台」变成「可感知的工作底座」。这不是铁律，是有节制的透明度。

## 异常处理

| 场景 | 触发条件 | 处理动作 |
|------|---------|---------|
| MOC 扫描无匹配 | 没有相关 MOC 覆盖当前话题 | 降级到路径 3（全文 grep 关键词） |
| BFS 遍历召回不足 | 路径 1 遍历 2-3 跳后语义筛选 < 5 张 | 扩大遍历到 4 跳；仍不足则启动路径 2 多 MOC 并行；再不足 → 标注「知识域覆盖不足」，建议后续编译时优先关注该领域 |
| 调用记录缺失 | `知识卡片调用记录.md` 不存在 | 标注「调用记录不可用」，仍正常使用卡片但不更新计数 |
| 链接网络有死链 | 死链检测发现卡片间链接断开 | 不阻断调用——死链是数据层体检的事，调用时跳过不可解析的链接 |
| 知识卡片目录为空 | `02-知识卡片/知识卡片/` 无文件 | 告知用户知识库未初始化，切换到无知识库模式回答 |
